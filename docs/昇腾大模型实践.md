# 基于昇腾生态的大模型训练实践报告

## 摘要

本报告系统总结了我们在华为昇腾（Ascend）AI 生态上开展的大模型训练实践。工作覆盖从 7B 规模模型的强化学习算法验证，到 32B 规模模型的监督微调（SFT）与离线强化学习（Offline RL），再到 405B 乃至万亿参数级 MoE 大模型的千卡/万卡集群预训练。依托昇腾 910 系列 NPU、CANN 异构计算架构以及 MindSpeed-LLM / MindSpeed-RL 等分布式训练框架，我们在数学推理、代码生成等关键基准上取得了业界领先结果，并完成了自研万亿参数 MoE 大模型 PCL-LLM-Model 的长稳预训练。此外，我们建设了开源评测框架 LLMEval，支撑了模型训练、数据蒸馏与效果评估的全流程。

## 1. 引言

### 1.1 背景

近年来，大语言模型（LLM）参数规模从数十亿迅速扩展到万亿级别，对计算基础设施、分布式训练框架和工程优化提出了极高要求。华为昇腾 AI 生态以昇腾 NPU 为核心，结合 CANN（Compute Architecture for Neural Networks）、PyTorch 昇腾适配层、MindSpeed-LLM / MindSpeed-RL 等软件栈，为大模型训练提供了国产化的高性能解决方案。

### 1.2 目标与范围

本报告旨在梳理我们在昇腾生态上的大模型训练实践，主要包括：

- 7B 规模模型的 GRPO 强化学习算法验证；
- 32B 规模模型的监督微调与离线强化学习；
- 千卡/万卡集群上 405B 至万亿参数级稠密/MoE 模型的预训练；
- 自研万亿参数 MoE 大模型 PCL-LLM-Model 的设计与训练；
- 美团 LongCat-Flash-Chat 模型的横向扩展实验；
- 开源评测框架 LLMEval 的建设与应用。

## 2. 昇腾大模型训练技术栈

### 2.1 硬件平台

- **昇腾 910B / 910C**：面向训练场景的高算力 NPU，支持 FP16/BF16/INT8 等多种精度。910C 采用双 Die 封装，节点内通过高带宽互联提升卡间通信效率。
- **超节点拓扑**：采用 **16 节点 × 8 卡** 的超节点基础单元，节点内 8 张 910C 通过 HCCS / UB 互联构建高速通信域。

### 2.2 软件栈

| 层级       | 组件                | 说明                      |
| ---------- | ------------------- | ------------------------- |
| 硬件       | 昇腾 910B/910C NPU  | 训练算力核心              |
| 计算架构   | CANN                | 算子编译、运行时、通信库  |
| 框架层     | PyTorch（昇腾适配） | 兼容主流生态              |
| 分布式训练 | MindSpeed-LLM       | 昇腾亲和的大模型训练框架  |
| RL 训练    | MindSpeed-RL        | 昇腾原生强化学习训练框架  |
| 并行策略   | TP / PP / EP / CP   | 张量/流水线/专家/序列并行 |

### 2.3 并行策略

针对大模型训练中的显存、通信和计算瓶颈，我们采用了混合并行策略：

- **TP（Tensor Parallel，张量并行）**：将单个 Transformer 层的参数切分到多张卡上，降低单卡显存占用。
- **PP（Pipeline Parallel，流水线并行）**：将模型按层切分到不同节点，扩展支持更深网络。
- **EP（Expert Parallel，专家并行）**：MoE 场景下将不同专家分配到不同卡/节点，是扩展专家数量的关键。
- **CP（Context/Sequence Parallel，序列并行）**：对长序列输入进行切分，降低激活值显存占用。

### 2.4 关键优化技术

- **昇腾亲和算子优化**：针对 NPU 的 Cube/Vector 单元特性，优化 Attention、MoE Router、FFN 等核心算子；使用 CANN 图编译与算子融合减少 kernel 启动开销。
- **通信优化**：节点内利用 910C 高带宽互联降低 TP/EP 通信延迟；节点间采用高效 All-to-All 与 All-Reduce 实现，保障 EP/DP 扩展效率。
- **显存与稳定性优化**：采用激活重计算、梯度检查点、ZeRO 优化器状态分片、异步 Checkpoint、故障自动恢复等技术，保障长稳训练。

## 3. 训练实践

### 3.1 RL 算法验证：7B 规模 GRPO 训练

在开展更大规模模型训练之前，我们首先在 7B 规模模型上验证了 GRPO（Group Relative Policy Optimization）等强化学习算法的正确性与有效性，为后续 32B 模型的 Offline RL 训练奠定了基础。

#### 3.1.1 基于 OpenRLHF 的 GRPO 复现

2025 年 4 月，我们基于 OpenRLHF 框架，在多个数学推理数据集上复现了 GRPO 算法的效果：

| 实验 | 基座模型 | 数据集 | 评测集 | 结果 |
| ---- | -------- | ------ | ------ | ---- |
| 实验 1 | Qwen2.5-7B-Base | GSM8K-train | GSM8K | **92.6%** |
| 实验 2 | Qwen2.5-7B-Base | OpenR1-Math-220K（筛选后） | Math500 | **80.0** |
| 实验 2 | Qwen2.5-7B-Base | OpenR1-Math-220K（筛选后） | AIME 2024 | **36.7** |
| 实验 3 | 鹏城脑海 2B | GSM8K-train | GSM8K | 从 59 提升到 **74.5**（+15.5） |

实验 2 的 AIME 2024 得分 36.7 在当时 7B 规模模型中名列前茅；实验 3 显示 GRPO 对小模型同样具有显著的提升效果。

#### 3.1.2 基于 MindSpeed-RL 的算法正确性验证

在 MindSpeed-RL 框架上，我们基于 Qwen2.5-Math-7B、Distill-Qwen-7B、OpenThinker3-7B 等多个模型进行了过饱和实验，验证了 GRPO 算法的正确性，并证明了持续 RL 训练对于提升推理正确率的重要性。

此外，我们基于 OpenThinker3-7B 模型，在 nvidia/AceReason-Math 数据集上使用 MindSpeed-RL 分别训练了 GRPO On-Policy 与 Off-Policy 配置。实验发现，只有在 On-Policy 配置下模型才能正常训练而不出现崩溃，这一结论为后续 Offline RL 的算法设计提供了重要参考。

### 3.2 32B 规模模型监督微调：PCL-Reasoner-V1

2025 年 7 月，我们基于昇腾硬件完成了 **PCL-Reasoner-V1** 模型的训练与发布。

- **基座模型**：Qwen2.5-32B-Base
- **训练数据**：开源 R1-0528 蒸馏数据集
- **训练方式**：监督微调（SFT）
- **训练平台**：昇腾 NPU 集群

经过微调，模型在数学推理能力上取得了显著提升：

| 评测集    | 准确率    |
| --------- | --------- |
| AIME 2024 | **85.7%** |
| AIME 2025 | **84.2%** |

在 32B 参数级别模型中，PCL-Reasoner-V1 表现稳居前列。我们已在 HuggingFace 和 ModelScope 开源模型权重与训练数据，并在启智社区和 GitHub 公开了完整训练代码与使用指导。

### 3.3 32B 规模模型离线强化学习：PCL-Reasoner-V1.5

2025 年 12 月，我们在 PCL-Reasoner-V1 基础上进一步完成了 **PCL-Reasoner-V1.5** 的训练。

- **基座模型**：PCL-Reasoner-V1
- **训练方法**：自研离线强化学习（Offline RL）
- **训练平台**：昇腾 NPU 集群

离线强化学习在避免在线采样高成本的同时，进一步激发了模型的推理潜力。实验结果显示：

| 评测集    | 准确率    |
| --------- | --------- |
| AIME 2024 | **90.8%** |
| AIME 2025 | **85.7%** |

在所有 32B 规模模型中，PCL-Reasoner-V1.5 实现了榜单第一。相关模型权重、训练数据与训练代码已开源，技术报告已在 arXiv 上发布。

### 3.4 千卡集群大规模训练实验

我们在千卡/万卡级昇腾集群上开展了多个超大规模模型的训练验证，重点评估了训练吞吐、MFU（Model FLOPs Utilization）和扩展效率。

| 模型             | 集群规模 | MFU        |
| ---------------- | -------- | ---------- |
| Llama3-405B      | 1024 卡  | **43.62%** |
| DeepSeek-V3-Base | 2048 卡  | 23%        |
| Kimi2-Base       | 2048 卡  | 20%        |

Llama3-405B 在 1024 卡昇腾集群上取得了 43.62% 的 MFU，表明昇腾平台在 400B 级稠密模型训练中具备较强的算力利用效率。

### 3.5 自研万亿参数 MoE 大模型：PCL-LLM-Model

#### 3.5.1 模型架构

**PCL-LLM-Model** 是鹏城自研的万亿参数级大模型，架构设计借鉴了 DeepSeek-V3 和 Qwen3，并针对昇腾硬件进行了亲和性优化。

| 属性           | 值                   |
| -------------- | -------------------- |
| **架构**       | MoE + GQA            |
| **路由专家数** | 128                  |
| 共享专家数     | 1                    |
| 专家分组       | 8                    |
| Top-K          | 2                    |
| **隐藏维度**   | 7168                 |
| FFN 维度       | 18432                |
| 专家维度       | 12288                |
| **网络层数**   | 32                   |
| Dense Layer    | 2                    |
| **GQA**        | attention heads = 64 |
| 词表大小       | 163840               |

#### 3.5.2 分布式训练配置

我们已在 **2048 卡（4096 Die）** 的昇腾千卡集群上成功完成了万亿参数级 MoE 大模型的预训练任务。

- **数据集规模**：5T tokens
- **训练时长**：30 天
- **集群拓扑**：16 节点 × 8 卡超节点基础单元，节点内 8 张 910C 高带宽互联
- **混合并行策略**：

| 配置项 | 训练配置 | 说明                                       |
| ------ | -------- | ------------------------------------------ |
| **TP** | 2        | Tensor Parallel（张量并行）                |
| **PP** | 8        | Pipeline Parallel（流水线并行）            |
| **EP** | 64       | Expert Parallel（专家并行）                |
| **CP** | 1        | Context Parallel（序列并行，可选 Ulysses） |

通过 TP=2、PP=8、EP=64 的组合，在 700+ 节点规模上实现了高效分布式训练。

### 3.6 LongCat-Flash-Chat 横向扩展实验

近期，我们基于美团 **LongCat-Flash-Chat** 模型开展了横向扩展实验，将 560B 参数规模的模型扩展至万亿参数级别。

- **扩展方式**：横向扩展（Width Scaling）
- **训练框架**：MindSpeed-LLM
- **硬件需求**：最少仅需 2 个超节点（256 Die）
- **精度保持**：推理精度与原 LongCat-Flash-Chat 模型一致，多个基准评测集上均能复现相同结果

该实验验证了昇腾生态在支持第三方开源大模型高效扩展训练方面的能力，证明了 MindSpeed-LLM 在 MoE/稠密混合架构上的灵活性与扩展性。

## 4. 评估基础设施：LLMEval

为支撑模型训练、数据蒸馏与效果评估，我们开发了大模型推理评估框架 **LLMEval**。该框架具有以下特点：

- **接口易用**：提供统一、简洁的评测接口，降低多模型、多数据集评估的接入成本。
- **性能稳定**：支持 vLLM、SGLang 等主流推理引擎，以及一键式多节点自动化推理评估。
- **任务覆盖广**：支持多种评测任务和数据集，覆盖数学推理、代码生成、通用能力等场景。
- **生态集成**：已全面支持项目组内部的数据蒸馏、数据清洗和模型评测等场景，显著提升了团队效率。

LLMEval 已在 Gitee 平台和启智社区开源，并获得平台认可和推荐。我们的 PCL-Reasoner-V1 和 PCL-Reasoner-V1.5 均基于该评测框架完成评测。

## 6. 总结与展望

本报告系统介绍了我们在昇腾生态上的大模型训练实践，覆盖了从 7B 模型 RL 算法验证、32B 模型微调到万亿参数 MoE 模型预训练的完整技术路径。主要成果包括：

- 在 7B 规模模型上验证了 GRPO 算法的正确性，为更大规模 RL 训练奠定了基础；
- 基于昇腾 NPU 完成了 PCL-Reasoner-V1/V1.5 的训练，在 AIME 等数学推理基准上达到 32B 级别领先水平；
- 在千卡/万卡集群上验证了 Llama3-405B、DeepSeek-V3-Base、Kimi2-Base 等超大规模模型的训练能力；
- 成功完成了自研万亿参数 MoE 模型 PCL-LLM-Model 的 5T tokens、30 天预训练；
- 基于 MindSpeed-LLM 实现了 LongCat-Flash-Chat 的万亿参数横向扩展，保持了原始模型精度；
- 建设了开源评测框架 LLMEval，支撑训练、蒸馏与评估全流程。

未来，我们将继续在以下方向深入：

- 进一步提升超长序列与超大规模 MoE 模型的训练效率；
- 优化 Offline RL 与多阶段后训练流程，提升模型推理与对齐能力；
- 推动昇腾训练框架与开源生态的深度融合，降低大模型训练门槛。
